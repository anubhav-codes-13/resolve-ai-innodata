01
GET /api/v1/admin/overview/stats
Powers: 5 top metric cards — Total Requests, Resolution Rate, Avg Handling Time, CSAT Score, Drop-Off Rate. Each card shows a main value and a trend badge with delta and direction arrow.
⚙ Both date_range and channel affect this response — stats values change per channel, trend deltas change per date range × channel. All 12 combinations must be supported.
Response · Example: date_range=7d, channel=All
JSON 
{
  "date_range": "7d",
  "channel":    "All",
  "stats": {
    "total_requests":    "28,431",    // pre-formatted string
    "resolution_rate":   "83%",
    "avg_handling_time": "2.3 min",
    "csat_score":        "92%",
    "drop_off_rate":     "11%"
  },
  "trends": {
    "total_requests":    { "delta": "+12.5%", "up": true  },
    "resolution_rate":   { "delta": "+4.2%",  "up": true  },
    "avg_handling_time": { "delta": "-0.4m",  "up": true  }, // lower = better
    "csat_score":        { "delta": "+2.1%",  "up": true  },
    "drop_off_rate":     { "delta": "+0.3%",  "up": false }  // higher drop-off = bad
  }
}
Field Notes
FieldTypeNotes
stats.* string Pre-formatted — render as-is. E.g. "28,431", "83%", "2.3 min"
trends.*.delta string Pre-formatted with sign and unit. E.g. "+12.5%", "-0.4m"
trends.*.up boolean true → green ↗ badge. false → red ↘ badge. Note: for avg_handling_time, a negative delta is still up: true because lower time = better performance.
02
GET /api/v1/admin/overview/ai-insight
Powers: the AI Insight card (blue-tinted panel with Sparkles icon) rendered below the filter row. Text is displayed with line-clamp-2.
Response · Example: date_range=7d, channel=All
JSON 
{
  "date_range": "7d",
  "channel":    "All",
  "insight":    "Support volume peaked on Friday at 7,200 interactions — a 24%
               surge vs. last week. Payment failure clusters drove 62% of escalations.
               AI resolution rate improved by 4.2 points to 83% after the SLM-v4.0
               hot-patch deployed Wednesday."
}
⚠ All 12 date_range × channel combinations must return a unique insight string. The frontend renders it verbatim — no HTML tags, no markdown.
03
GET /api/v1/admin/overview/channel-trends
Powers: the "Customer Call Trends" 3-line Recharts chart.
Resolution % → blue solid line → right Y-axis (0–100%)
Volume → emerald dashed line → left Y-axis (raw count, formatted ≥1k as "Nk")
Escalation % → amber dashed line → right Y-axis (0–100%)
Response · Example: date_range=7d, channel=All
JSON 
{
  "date_range": "7d",
  "channel":    "All",
  "data": [
    { "time": "Mon", "resolution": 78, "volume": 5200, "escalation": 16 },
    { "time": "Tue", "resolution": 81, "volume": 4900, "escalation": 14 },
    { "time": "Wed", "resolution": 74, "volume": 6100, "escalation": 22 },
    { "time": "Thu", "resolution": 83, "volume": 5600, "escalation": 12 },
    { "time": "Fri", "resolution": 70, "volume": 7200, "escalation": 28 },
    { "time": "Sat", "resolution": 86, "volume": 4100, "escalation": 10 },
    { "time": "Sun", "resolution": 90, "volume": 3200, "escalation": 8  }
  ]
}
Field Notes
FieldTypeNotes
time string X-axis label.
24h → 8 points: "00:00" … "22:00"
7d → 7 points: "Mon" … "Sun"
30d → 4 points: "Week 1" … "Week 4"
resolution int 0–100 Percentage value. No % sign. Recharts right Y-axis (yAxisId="pct").
volume int Raw count. No formatting. Recharts formats ≥1000 as Nk on the left Y-axis (yAxisId="vol").
escalation int 0–100 Percentage value. No % sign. Recharts right Y-axis (yAxisId="pct").
04
GET /api/v1/admin/overview/issue-clusters
Powers: the Issue Clusters panel (right side of chart row). Each row shows: topic name · volume count · change % · 8-point micro sparkline. Frontend paginates client-side at 5 per page.
✓ Do not paginate server-side. Return the full filtered list for the given date_range × channel. The frontend slices it.

FieldTypeNotes

05
POST /api/v1/admin/overview/ask
Powers: the follow-up AI question input bar at the bottom of the screen. The "Ask" button is disabled when the input is empty.
Request Body
JSON 
{
  "question":   "Why did escalations spike on Friday?",  // required
  "date_range": "7d",                                    // active filter at submit time
  "channel":    "All"                                     // active channel filter
}
Response
JSON 
{
  "answer": "Friday escalations reached 28% primarily due to a payment gateway
             degradation between 14:00–19:00 affecting Card Declined and Payment Failed
             clusters. Volume surged 24% above weekly average."
}
ℹ The date_range and channel context must be used by the AI backend to scope the answer to the correct data window. The answer string is rendered verbatim in the UI.
